# 사주 MVP Product Brief

## 1) Problem
- 사주는 여전히 수요가 크지만, 기존 서비스는 광고 과다, 신뢰도 부족, UX 복잡성 문제가 큼.
- 초심자는 "어떤 입력이 필요한지"부터 막히고, 결과는 난해하거나 길기만 해서 행동으로 이어지지 않음.
- 모바일 중심 사용자가 많은데 웹/앱 경험이 일관되지 않은 경우가 많음.

## 2) Target Users
- 20~39세 모바일 중심 사용자
- "무료로 빠르게 내 사주를 보고 싶은" 입문자
- 연애/진로/재물 같은 주제별 해석을 가볍게 소비하는 사용자
- 카카오톡/인스타 공유를 통해 결과를 전파하는 소셜 사용자

## 3) Core Features (MVP)
- 무료 사주 입력 폼: 이름, 생년월일/시간, 성별, 양력/음력
- 즉시 결과 카드: 오늘의 운세 요약 + 성향/강점/주의점(현재는 mock)
- 결과 저장(익명 세션 기준) 및 재조회
- 웹/모바일 공통 입력 경험
- 추후 유료 확장을 고려한 결제/계정 구조 준비

## 4) Monetization Hypothesis
- Freemium 구조
- 무료: 기본 사주 결과(요약)
- 유료 단건(₩3,900~₩9,900): 상세 리포트(PDF), 궁합/택일/질문형 추가 해석
- 구독(월 ₩7,900~₩14,900): 월간 운세, 주제별 심층 해석, 광고 제거
- B2B 확장: 콘텐츠 제휴 API(미디어/커뮤니티)

## 5) 2-Week MVP Scope
- Week 1
  - 모노레포 초기 세팅 (web/mobile/api/shared)
  - 무료 사주 입력 UI (웹/모바일)
  - mock 결과 API 및 폼 연동
- Week 2
  - 결과 카드 고도화(문구 템플릿, 로딩/에러 UX)
  - 기본 분석 이벤트 설계(page_view, form_submit, result_view)
  - 배포 파이프라인 초안(Vercel + Render/Fly)
  - 다음 단계(실제 사주 엔진/LLM 결합) 위한 기술 스펙 확정

## Out of Scope (MVP)
- 정교한 만세력 계산 엔진
- 결제 실구현(연동 계획만 문서화)
- 소셜 로그인 실구현
- 대규모 추천/개인화
