# GTM_CONVERSION_PLAYBOOK

사주 웹-first MVP의 초기 매출/전환 최적화 실행안입니다.

## 1) 전제
- 타깃: 10~30초 소비, Instagram/Kakao 공유 중심 사용자
- 톤: 전문가형, 사실형, 확률형 문장
- 목적: 무료 결과 소비 후 유료 전환(표준/심화)

## 2) 가격 실험 (4주)
- 실험 A: `표준 3,900원 / 심화 9,900원`
- 실험 B: `표준 4,900원 / 심화 12,900원`
- 실험 C: `표준 2,900원 첫구매 / 심화 9,900원`
- 성공 기준: 결제 전환율 + 객단가 + 환불률 동시 평가
- 세그먼트: 신규/재방문, 유입 채널(인스타/카카오/직접)

## 3) 페이월 전략
- 무료: 요약 + 핵심 1개 섹션만 완전 공개
- 표준: 과거/현재/가까운 미래 + 분야별 요약 + 실행 가이드
- 심화: 표준 포함 + 시나리오 분석 + 90일 액션 플랜
- 페이월 카피 원칙
  - 가치 중심: `당장 쓸 수 있는 실행 가이드 제공`
  - 공포 자극 금지: `놓치면 손해`류 문구 제한
  - 신뢰 강조: `확률 기반 해석`, `과장 없는 리포트`

## 4) 공유 카드 전략
- 프리셋 3종
  - 오늘의 한줄: 즉시 공유용, 20~28자
  - 성향핵심: 자기소개형, 저장/재공유 유도
  - 주의포인트: 행동 변화 유도형
- 카드 CTA: `내 상세 해석 보기` 단일 문구 유지
- 채널 최적화
  - Instagram: 짧은 카피 + 해시태그 2개 이하
  - Kakao: 문장형 1~2줄, 링크 우선

## 5) 퍼널 지표
- 노출: 랜딩 방문수, 유입 채널 비중
- 활성: 무료 입력 완료율, 결과 도달율
- 전환: 상품 클릭률, 결제 시도율, 결제 완료율
- 확산: 공유 버튼 클릭률, 공유 후 재유입율
- 품질: 환불률, CS 문의율, 재구매율

## 6) A/B 테스트 계획
- 테스트 1: 결과 페이지 CTA 위치(상단 고정 vs 하단 고정)
- 테스트 2: 잠금 미리보기 길이(2문단 vs 4문단)
- 테스트 3: 공유 프리셋 기본값(오늘의 한줄 vs 성향핵심)
- 테스트 4: 가격 앵커 표시(심화 먼저 제시 vs 표준 먼저 제시)
- 운영 원칙
  - 최소 1주/테스트 유지
  - 동시 다중 테스트는 최대 2개
  - 통계 유의성 미달 시 보수적 해석

## 7) 주요 리스크와 대응
- 신뢰 리스크: 과장 카피로 이탈
  - 대응: 확률형 문장 규칙 강제, 내부 검수 체크리스트 운영
- 가격 저항: 초기 유료 전환 저조
  - 대응: 첫구매 가격 실험 + 표준 진입가격 최적화
- 공유 저조: 카드 문구 매력 부족
  - 대응: 프리셋 지속 실험, 사용자 문구 저장 기능 추가
- 법/정책 리스크: 플랫폼 정책 충돌 가능성
  - 대응: 건강/법률/투자 확정 조언 금지 문구 명시

## 8) 실행 우선순위
1. 결과 페이지 잠금 미리보기 + 상품 CTA 배치
2. 공유 카드 프리셋/복사 기능 구현
3. 이벤트 로깅(퍼널 지표 수집)
4. 가격/카피 A/B 실험 론치
